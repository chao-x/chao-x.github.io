Docker入门

Docker是采用写实复制的模式，如果想修改一个文件，会先从读写层下面的只读层复制到读写层，该文件的只读版本仍然存在，但是已经被读写层的文件副本所隐藏。

启动容器

docker run -i -t --name "ubuntu-system" ubuntu /bin/bash
  • 需要root权限时要加上sudo开头
  • -i 表示 容器中的STDIN是开启的
  • -t 表示 为容器分配一个伪tty的终端
  • –name 唯一标识容器名字 只允许 : a-z A-Z 0-9 _ . -
  • –restart 可以使用该命令自动重启容器当发生某种错误
    • –restart=always，无论退出代码是什么都会重启
    • –restart=on-failure，只有退出代码不为0才会重启
    • –re start=on-failure:5，限制重启次数

查看容器

docker ps -a
  • -a 表示查看所有容器，包括已经停止的 【 -all 】
  • -l 表示查看最后一次运行的容器 【 -laste 】
docker start ubuntu
docker stop ubuntu
  • 启动已经停止的容器
  • 停止一个容器

查看容器内部运行的进程

docker top ubuntu

查看容器内部日志

docker logs -f ubuntu_code_2
  • 显示实时日志
  • -t 显示时间 【 time 】

查看容器详细信息

docker inspect ubuntu-system

附着容器

docker attach ubuntu
  • 我们启动了之前停止的ubuntu容器后是不能直接进入/bin/bash的
  • 使用 attach 可以附着到已经启动的容器上

守护式容器

docker run --name ubuntu-code -d ubuntu /bin/sh -c "while true;do echo hello world;sleep 1;done"
  • -d 分离的，在后台运行 【 detached 】
  • -c 运行的代码 【 code 】

在容器内部运行进程

docker exec -d ubuntu-system touch /etc/new_config_file
  • 创建一个文件
docker exec -t -i ubuntu-system /bin/bash
  • 开启一个新终端

删除容器

docker rm ubuntu-system
docker rm `docker ps -a -q`
  • -q 只显示容器的ID 【 -quiet 】

Docker镜像

查看镜像

docker images
docker images ubuntu
  • 只看指定的镜像
docker history image-name
  • 查看镜像的构建过程

拉取镜像

docker pull ubuntu

指定镜像版本

docker run -i -t  --name ubuntu-system ubuntu:12.04 /bin/bash

查找镜像

docker search ubuntu

Commit方式构建镜像

docker login
  • 需要登陆docker hub
docker commit -m="first-test-with-ubuntu" --author="chaox" ubuntu-system wallace2504/ubuntu-test
docker commit -m="first-test-with-ubuntu" --author="chaox" ubuntu-system:system wallace2504/ubuntu-test
  • -m 备注
  • : 指定标签，后面的标签设置都类似此方式

Dockerfile方式构建镜像

# Version 0.0.v1
From ubuntu:14.04
MAINTAINER Author Name "Author's Email"
RUN apt-get update
RUN apt-get install -y nginx
Run echo 'Hi I am in your container'\
		> /usr/share/nginx/html/index.html
EXPOSE 80
  • Docker会从基础镜像生成一个容器，执行一条执行后，对容器作出修改生成一个新的镜像层（类似commit操作），再基于刚刚的新镜像生成新容器，以此类推。
  • 可以看出，当某条指令失效时，返回的镜像依旧是可用的镜像。
  • RUN [“apt-update”,“update”] ，不支持shell或不希望用shell，可以使用exec格式
docker build -t="chaox/nginx-1" .
  • 注意需要加路径【 . 】
  • –no-cache 略过缓存
# Version 0.0.v1
From ubuntu:14.04
MAINTAINER Author Name "Author's Email"
ENV REFRESHED_AT 2020-11-10
RUN apt-get update
EXPOSE 80
  • 使用ENV设置环境变量，使用该模版时，仅需改变时间，后面的指令无须依赖缓存
docker run -d -p 80 --name ubuntu-sysy chaox/nginx-1 nginx -g "daemon off;"
  • 需要安装nginx， -g “daemon off” 使得nginx前台运行
  • 需要指定端口

前台运行和后台运行的差异（存疑）

1.前台是可见可交互的，后台是不可见不可交互

2.前台会占用大量的系统资源，后台适合占用资源小的

3.前台类似我们的网页，游戏。后台类似我们的网络进程

Docker 端口映射

docker ps -l
docker port ubuntu-sys 80
  • 查看映射的端口信息

指定映射关系

docker run -d -p 127.0.0.1:8080:80 --name nginx-test chaox/nginx-1  nginx -g "daemon off;"
  • 宿主机8080端口映射到容器80端口
  • -P 可以公开在Dockerfile中指定的端口

推送到DockerHub

docker push chaox/nginx-1

删除镜像

docker rmi chaox/nginx-1
docker rmi `docker images -q`
