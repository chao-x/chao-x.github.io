《MySQL必知必会》学习笔记一

《MySQL必知必会》学习笔记一

1.SQL概述

  • SQL("Structured Query Language")，结构化查询语言

  • SQL可以说是一种脚本语言，因为它是由解释器来执行的

  • 主键，唯一标识一行，可以由一列或多列组成

    • 主键不能为空
    • 最好不更新主键的值
    • 最好不重用主键的值（如果删除了主表的某个主键，但没有删除从表对应的行，如果重用，可能会有错误的关联）
    • 主键不要使用容易更改的值

2.MySQL概述

  • DBS=DSB+DBMS，数据库系统=数据库+数据库管理系统

  • MySQL是一种DBMS，是基于C/S架构即分为客户机和服务端模式。

  • MySQL不区分大小写

  • 所有关于数据、数据增删改查都由服务端来完成

  • DBMS为服务端软件，客户端可以是MySQL提供的工具，包括脚本语言，程序设计语言

  • 登录

    • mysql -u root -p

  • 默认端口：3306

3.MySQL命令

注意

  • 每个命令（最好）需要以分号来表示结束
  • 多条语句必须以分号来表示结束

展示所有数据库

SHOW DATABASES

选择数据库

USE database-name

展示某个数据库的所有表

  • 需要选择数据库
SHOW TABLES

展示某个表的所有列及其属性

SHOW COLUMNS FROM table-name
  • 快捷方式
DESCRIBE table-name

展示错误或警告信息

SHOW ERRORS

SHOW WARNINGS

4.SELECT初级命令

一.SELECT基础

简单的SELECT命令

SELECT order_num,cust_id
FROM orders;

检测出所有行

SELECT *
FROM orders;

检测出不同的行

  • 不能指定部分列，使用DISTINCT关键字后，SELECT的所有列都会被要求不同（组合方式），单列可以有相同的内容
SELECT DISTINCT prod_id,vend_id
FROM products;

返回指定的数量

  • 不能多于指定的行数
SELECT prod_id
FROM products
LIMIT 5;
  • 从指定的行数开始返回指定的数量
  • 注意第一行的行号为0
  • 下面这个语句表示从第六行开始返回五行
SELECT prod_id
FROM products
LIMIT 5,5;

使用完全限定名称

  • 完全限定列名
  • 完全限定表名
SELECT products.prod_id
FROM formysql.products;

二.ORDER BY排序

  • 关键字：ORDER BY

  • 必须在WHERE和FROM后面

  • 默认不分区大小写，A和a是一样

  • 命令包含字句组成，SELECT中的FROM就是一个字句，同理ORDER BY也是

升序排序

SELECT prod_id,prod_name
FROM products
ORDER BY prod_id;

多个列排序

  • 按顺序，先按第一个再按后一个排序
SELECT prod_id,prod_name
FROM products
ORDER BY prod_id,pro_name

指定排序顺序

  • 默认为升序排序，关键字为ASC
  • 可以使用降序排序，关键字为DESC
  • 下面语句先根据id降序排序，再根据name升序排序
  • 想要在多个列降序排序，必须为每个列使用DESC关键字
SELECT prod_id,prod_name
FROM products
ORDER BY prod_id DESC,prod_name;

和LIMIT配合使用

  • LIMIT必须在ORDER BY后面使用
  • 注意当使用来分页时，如果相同的列会随机返回，需要注意使用
SELECT prod_id,prod_name
FROM products
ORDER BY prod_id DESC,prod_name
LIMIT 5,5;

三.WHERE过滤数据

  • WHERE关键字
  • 操作符：=，<>或者=!，<，>，<=，>=，BETWEEN，IS NULL
  • 字符串必须使用引号
  • 如果使用ORDER BY关键字，必须在WHERE后面
SELECT prod_id,prod_name,prod_price
FROM products
WHERE prod_price = 2.5;

范围筛选BETWEEN

SELECT prod_id,prod_name,prod_price
FROM products
WHERE prod_price BETWEEN 2 AND 3;

空值检查 IS NULL

SELECT prod_id,prod_name,prod_price
FROM products
WHERE prod_price IS NULL;

AND组合WHERE字句

  • 可以使用括号防止混淆
SELECT vend_id,prod_id,prod_name,prod_price
FROM products
WHERE (prod_price BETWEEN 2 AND 5) AND vend_id=1003;

OR组合WHERE语句

  • AND的计算优先级是比OR高的
  • 可以使用括号改变优先级
SELECT vend_id,prod_id,prod_name,prod_price
FROM products
WHERE vend_id=1002 OR vend_id=1003 AND prod_price>10;

等价于

SELECT vend_id,prod_id,prod_name,prod_price
FROM products
WHERE vend_id=1002 OR (vend_id=1003 AND prod_price>10);

IN 指定具体范围

  • 需要配合括号一起使用
  • IN操作符比OR操作更快
  • IN可以配合SELECT子语句一起使用
SELECT vend_id,prod_id,prod_name,prod_price
FROM products
WHERE vend_id IN (1002,1003)
ORDER BY vend_id;

NOT 否定

  • MySQL支持NOT对IN，BETWEEN，EXISTS取反
SELECT vend_id,prod_id,prod_name,prod_price
FROM products
WHERE vend_id NOT IN (1002,1003)
ORDER BY vend_id;

四.通配符

LIKE 模糊匹配

  • 必须和WHERE一起使用
  • 通配符必须和LIKE一起使用，使用LIKE关键字配合通配符可以进行模糊匹配
  • 通配符必须在引号中使用
  • 搜索模式，由字符串和统配符组成，”j%e“即为在搜索模式中

% 任意字符出现任意次数

  • 类似于正则表达式的*
  • 可以匹配0个字符
  • 通配符必须在引号中使用，不管是什么类型
SELECT vend_id,prod_name
FROM products
WHERE vend_id LIKE '%0%';

_ 匹配单个字符

  • 必须且只能匹配一个字符
  • 如果匹配1003则第二个SQL语句无法匹配到
SELECT vend_id,prod_name
FROM products
WHERE vend_id LIKE '10_3';

SELECT vend_id,prod_name
FROM products
WHERE vend_id LIKE '100_3';

通配符的代价

  • 使用时间相比较其他匹配较长
  • 尽量不要把通配符放在最开始处，如"%003"，否则会花费大量时间

五.正则表达式

REGEXP 关键字

  • REGEXP意思是正则表达式

  • 和LIKE关键字一样，正则表达式必须和REGEXP一起使用

  • 不加任何符号，表示包含即可

SELECT vend_id,prod_name
FROM products
WHERE prod_name REGEXP 'brid';
  • 和LIKE的区别，LIKE需要和整个串匹配，REGEXP包含即可

. 匹配任意字符

SELECT vend_id,prod_name
FROM products
WHERE prod_name REGEXP 'brid.';

错误示范：

SELECT vend_id,prod_name
FROM products
WHERE prod_name REGEXP 'brid.';
  • 这样写其实是等价于 REGEXP 'brid'
  • 该字符最好和位置字符搭配使用

OR 或匹配

SELECT vend_id,prod_name
FROM products
WHERE prod_name REGEXP 'brid|anvil|pack';

错误示范：

SELECT vend_id,prod_name
FROM products
WHERE prod_name REGEXP 'brid|anvil|pack c';
  • 这样是不能匹配anvilc的，匹配的是pack c

也可以使用

SELECT vend_id,prod_name
FROM products
WHERE prod_name REGEXP '[ab]c';

或者等价于

SELECT vend_id,prod_name
FROM products
WHERE prod_name REGEXP '[a|b]c';

错误示范：

SELECT vend_id,prod_name
FROM products
WHERE prod_name REGEXP '[brid|anvil]c';
  • 上面是无效的，不能匹配bridc
  • []只能匹配里面的一个字符,就是说上面等价于[bridanvil]c

范围匹配

SELECT vend_id,prod_name
FROM products
WHERE prod_name REGEXP '[1-9]';

转义字符

  • 当想要匹配 . 时，需要使用转义

  • 使用 \\ 表示转义，如：

SELECT vend_id,prod_name
FROM products
WHERE prod_name REGEXP '\\.';

特殊符号类

示例一：

SELECT vend_id,prod_name
FROM products
WHERE prod_name REGEXP '[[:digit:]]{4}';
  • 注意这里[:digit:]必须再加一层括号，因为不加括号实际为0123456789{4}

示例二：

  • 之前匹配的都是要满足至少一个

  • 如匹配[0-9] stick

SELECT vend_id,prod_name
FROM products
WHERE prod_name REGEXP '[0-9] stickz?';

定位符

示例：

SELECT vend_id,prod_name
FROM products
WHERE prod_name REGEXP '^[0-9]';
  • ^可以在否定一个集合，这里存在疑问

不用数据库使用正则表达式

SELECT 'HELLO'
REGEXP 'H'

六.函数的使用

Concat 拼接字段

SELECT Concat(vend_name,'(',vend_country,')')
FROM vendors
ORDER BY vend_name;

Trim 去掉空格

  • Trim 去掉作用两边的空格

  • LTrim 去掉左空格

  • RTrim 去掉右空格

SELECT Concat(Trim(vend_name),'(',vend_country,')')
FROM vendors
ORDER BY vend_name;

AS 使用别名

SELECT Concat(Trim(vend_name),'(',vend_country,')') AS titile
FROM vendors
ORDER BY vend_name;
  • 如果客户机需要一个不能函数空格的列名，而该表的列名又有空格，可以使用别名

算术计算

SELECT quantity,item_price,quantity%item_price AS expanded_price
FROM orderitems;
  • 支持基本的算术计算，包括取余%
  • 特殊用途：
SELECT 8*8;

SELECT Now();

转换大小写

  • Upper()转换大小
  • Lower()转换小写
